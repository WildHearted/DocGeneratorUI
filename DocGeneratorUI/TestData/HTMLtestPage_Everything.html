​Introduction to Content Authoring

Most of the time, it seems straight forward to capture text, images and tables into documents like in MS Word, but with the Service Design and Development Portfolio (SDDP)System, things works a little different. The SDDP is an online system that is build/based/hosted as a SharePoint solution. Capturing content in an online system is a little different to writing/authoring MS Word documents, although the principles are the same.

In the SDDP system, some content is captured as plain text fields (words without any formatting) and other content is captured as Enhanced Rich Text (ERT) content. Enhances Rich Text content are in essence words (text) but iy includes formatting, tables and images/pictures and other forms of media as well.

Depending on selections via the user interface (UI), the Sharepoint content is processed by a server service application (the DocGenerator), and automatically author various MS Word and MS Excel documents.

Plain Text

SDDP store some content as Plain Text fields, which is pure text that doesn't contain any formatting or other types of artifacts. Therefore, there are very seldom any complications with Plain Text conteant.

Enhanced Rich Text

Enhanced Rich Text (ERT) fields are stored as HTML. As you may know, HTML (HyperText Mark-up Language) content can consist of text, formatting information and other artifacts like images, tables etc. The formatting indicates how the text and other artefacts are formatted and presented to the user. Most internet web sites are HTML which the browser you use, translates and displays as the web page you see on a monitor. The format “instructions” in HTML consist of a defined set of HTML tags. For example <p>My text.</p> means that “My Text.” Is a separate paragraph. The tag <H1>My Heading</H1> is a separate Heading level 1 paragraph.

Each ERT field contains the text and the various HTML tags that indicate how the text is formatted. The DocGenerator needs to read, the HTML content, interpret it and then construct the relevant documents and/or workbooks. The DocGenerator reads the HTML contained in an ERT field, then deciphers the HTML to determine the format of the text and each artefacts contained in the content. If the contents is valid and it contains sufficient format information for the DocGenerator to include it in the document, it translates the HTML into an Open XML format, which is the basis of all Microsoft Office and Adobe Acrobat documents. 

When the DocGenerator cannot determine how the author intended to format the text and artefacts (tables or images) based on the HTML, it will indicate that in the “generated” document contains content that it cannot be interpreted to produce a properly formated document.

Therefore, the purpose of this document is to provide you with a guideline, directions and tips on how to author valid and properly formatted content that the DocGenerator will be able to read and properly translate when it produces a document or workbook.
